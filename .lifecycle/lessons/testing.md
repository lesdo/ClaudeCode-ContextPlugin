# Testing Lessons

- [2026-07-04] 先建测试框架再重构：153 tests 让 Phase B/C/D 存储迁移零回归。测试套件 5 秒跑完全量，每次提交前自动验证。
- [2026-07-04] bash `while read` 管道在右侧时创建子 shell，pass/fail 计数器在父 shell 不更新。用临时文件 + `< "$TMPFILE"` 重定向替代。
- [2026-07-08] **"工具可发现" ≠ "工具可用"** — MCP 注册成功 + 工具出现在列表只是连通性验证，数据通路正确性必须端到端：调用工具 → 检查返回值 → 确认数据源是正确 DB。`session_list` 返回 `[]` 不是空库，是连错了库。
- [2026-07-08] **`exit 0` 是静默杀手** — 脚本提前退出且返回码为 0 时，上游 hook 系统感知不到任何异常。关键路径的早期退出必须有显式告警或改用非零退出码。本次 `session-start.sh` 的 `exit 0` 让 MCP 初始化、崩溃检测、简报注入全部静默跳过。
- [2026-07-12] **测试路径必须与生产路径一字不差** — `test_compact.sh` 检查 `.claude/context/briefing/` 但 hook 写入 `.context/briefing/`，导致 3 个测试全部失效且长期未被发现。路径差异哪怕一个目录层级，测试都在验证错误的东西。自检方法：测试中引用的路径变量必须来自与生产代码同一个源头（如 `_common.sh` 的变量），禁止硬编码重复。
- [2026-07-12] **`assert_contains` 签名是 `(label, FILE, pattern)` 不是 `(label, STRING, pattern)`** — 第二个参数被 `grep` 当作文件路径打开。传字符串值时必须用进程替换 `<(echo "$VAR")`。API 合约必须读实现而非凭名称猜测。
